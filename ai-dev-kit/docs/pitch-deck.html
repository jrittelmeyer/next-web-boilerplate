ai-dev-kit · v0.3.0

A portable operating method for agentic development.

Six skills, three hooks, and nine techniques that turn AI-assisted building into a repeatable pipeline — token-lean, verification-first, and installable into any project in one command. Distilled from shipping a production monorepo to a verified 100/100.

node ai-dev-kit/install.mjs --adapter adapters/<project>.json --global --hooks

01 · The pipeline

Every unit of work runs the same loop.

orient→ plan-gate→ dep-check→ build→ live-verify→ review / simplify→ checkpoint→ periodic audits

Highlighted stages are kit skills; the rest are conventions (the playbook) or Claude Code built-ins the kit composes with. Each stage catches the failure class the previous one can't — wrong direction, supply-chain surprises, compiles-but-doesn't-work, context exhaustion.

02 · The skills

Six skills carry the mechanics.

checkpoint

step boundary

Commit + push, judge context health, then continue — or hand off with a paste-ready resume prompt that spares the next session an hour of re-derivation.

composes → tidy (cache prune) · records plan-approval state in memory

dep-check

auto-nudged

Registry-verify before any dependency change: real version, dist-tags, release-age window, pin policy, peers, standing holds. Never versions from blog posts or model memory.

hook fires on → package.json edits · pm add/update/install

live-verify

auto-nudged

The gate proves it compiles; only driving the flow proves it works. Fresh prod build on a dedicated port, exercise the actual change, report verbatim evidence.

hook fires before → every git commit · precedes checkpoint

project-audit

periodic

Score every feature group /100 against the best-available bar, verify docs ↔ code, and turn every lost point into a prioritized backlog row.

feeds → backlog · doc-audit · git-bounded between passes

doc-audit

periodic

Keep docs, agent context, memory, and showcase sets accurate and token-lean. Fix drift, slim the hot path, archive instead of delete, measure the savings.

dual-home → installs globally too · consumes project-audit reports

tidy

on demand

Prune the unbounded local build cache; surface — never auto-run — the judgment-required cleanups: orphaned dev servers, stale e2e users, dangling images.

invoked from → checkpoint (deeper pass) · pre-push backstop

03 · The hooks

Automation that advises, never blocks.

HandlerFires onInjects
dep-check-nudge package.json edits; pnpm add-class commands "registry-verify before relying on it"
live-verify-reminder any git … commit, compound commands included "confirm the live loop ran for product source"
skill-drift-guard direct edits to installed skills/hooks "edit the kit source and re-install instead"

Policy

Hooks inject one sentence of context; the agent decides. A wrong advisory costs a sentence — a wrong block halts legitimate work and trains bypass. Handlers are pure-Node stdin→stdout scripts (Windows-safe, no jq), drift-guarded like skills. All three proven firing live on day one. The rejections are recorded too: no Stop-hook nag, no cache hook — existing cadence covers both.

04 · The playbook

Nine techniques that don't fit in a skill.

The lifecycle pipeline

One loop for every unit of work; skip a stage only out loud.

Plan → sign-off → build

Wrong direction is the most expensive failure; a plan costs a paragraph.

Context-tier discipline

Token spend follows load frequency — the always-loaded tier is paid every turn.

Memory discipline

One durable, non-derivable fact per file; an index loaded each session.

Cheapest sufficient probe

Grep before read, registry before web, diff-bounded audits.

Fan-out research

Broad sweeps go to a subagent; only the conclusion returns.

The archive pattern

Move detail off the hot path verbatim — preserve everything, pay for nothing.

Resume prompts

Five parts, findings marked verified-where; a paragraph replaces an hour.

Automation review

Advise-never-block; record the rejected hooks alongside the shipped ones.

05 · Portability

Generic skills. One adapter file. Any project.

Skill bodies never hardcode a project — package manager, gate commands, ports, doc paths all live in one adapter config the skills read at run time. Missing fields degrade gracefully: the skill derives from the repo and says so.

  1. Write your adapter — or start from the reference (adapters/next-web-boilerplate.json, schema-validated, every field optional).
  2. Run node ai-dev-kit/install.mjs --adapter … --global --hooks — byte-identical skill installs, hook handlers, idempotent settings merge that touches only kit-owned entries.
  3. Keep honest with install.mjs --check — exit 1 names any installed file that drifted from kit source. Edit the kit, re-install; never the installed copies.

06 · Where this goes

Versioned here, extractable anywhere.

The kit is semver'd (VERSION · CHANGELOG.md · machine-readable manifest.json with the composition graph) and has no dependency on its host repo. Next: extraction to a standalone repository, imported here and into any similar project by the same one-command install. The method travels; only the adapter is yours to write.

The pitch, in one line

Quality as the default, tokens spent where they buy confidence, and a pipeline that reminds itself — so every session starts oriented and ends verified.
